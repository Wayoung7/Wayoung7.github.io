CSAPP深入理解计算机系统：数据在计算机中的表示

发表于2024-07-21|更新于2024-07-28|笔记
|阅读量:

CSAPP是所有计算机专业的学生和程序员必读的书，我自己直到读了这本书后，才认识到之前对于C语言和计算机的认识有多么浅薄。因此我想把自己读完这本书和听完CMU-213的公开课后的理解写成笔记分享给大家。

注：本篇文章默认读者已了解基本的数据单位（bit, byte）,位运算以及不同进制之间的转换，因此不再对这些概念进行解读

我们可以暂且认为数据在计算机中的表示方式为一个由0和1组成的数组。

位运算（Bitwise Operations）

C语言中基本的位运算有& | ~ ^ << >>。我想重点讲一下移位操作，

左移（Left Shift）<<

  • 把所有位向左移动x格，丢掉左边溢出的位，右边用0补全，比如 01100010 << 3 == 00010000

右移（Right Shift）>>

  • 所有位向右移动x格，丢掉右边溢出的位，左边补全有两种方式：
    • Logical Shift：左边用0补全。
    • Arithmetic Shift：用原先左边最高位补全，比如 10100010 >> 3 = 11110100, 01100010 >> 3 == 00001100。

C语言中对于有符号整数，右移使用Arithmetic Shift，无符号整数右移使用Logical Shift。

移位的距离小于0或大于等于一个字长（32位系统为4 byte，64位系统为8 byte）是未定义的。在大多数系统上，移位距离大于等于一个字长时，计算机会先把移位的距离模上字长，然后再移位，比如在32位系统中:

整数（Integer）

C语言对于有符号整数和无符号整数都定义了相应长度的数据类型。

无符号整数（Unsigned Integer）

无符号整数可以直接转换为二进制，比如用8 bit可以表示 282^828 个数，即 00 00 00 00 到 11 11 11 11 (0-255)。对于一个w位的unsigned int，能够表示的范围是 000-2w−12^w-12w−1，我们记他的最大值为 UMaxwUMax_wUMaxw​（每一位都为1）。每一个在范围中的数都有唯一的 Binary Number 与其对应。

有符号整数（Signed Integer）

绝大多数的机器都采用补码 Two’s Complement Encoding 的方式表示有符号整数，即最高位的权重变为 −2w−1-2^{w-1}−2w−1 （称为 sign bit），其它位的权重不变。对于一个 www 位的有符号整数：

  • 正数
    • 0为所有位都为0 (0000)
    • 最大的正数为最高位为0，其余位为1 (0111)，记为 TMaxw=2w−1−1TMax_w = 2^{w-1}-1TMaxw​=2w−1−1
  • 负数
    • 最小的负数为最高位为1，其余位为0 (1000)，记为 TMinw=−2w−1TMin_w = -2^{w-1}TMinw​=−2w−1
    • 最大的负数（-1）为所有位都为1 (1111)

对于一个用补码表示的有符号整数，sign bit 为0即为正数，1即为负数。

以下是不同长度的整数能表示的范围：

  • ∣TMin∣=TMax+1|TMin| = TMax + 1∣TMin∣=TMax+1 （有符号整数的表示范围并不是关于原点对称的）
  • UMax=2∗TMax+1UMax = 2 * TMax + 1UMax=2∗TMax+1

使用补码的好处

  • 0只有一种表示方式，没有重复
  • 有符号和无符号整数的加减操作是统一的
  • 取一个数的负数只需翻转所有bit并加1

不同长度的整数之间的转换

扩展（Expanding）：增加长度

  • 对于无符号整数，直接在前面补上0，不改变数值。（Zero Extension）
  • 对于有符号整数，在前面补上原本最高位的值，可以自行证明同样也不会改变原先的数值。（Sign Extension）

截断（Truncating）：减少长度

  • 对于无符号整数和有符号整数，直接丢掉溢出的位，对于较大的数数值可能改变，对于有符号整数其正负性也可能会改变。

整数运算

加法

通过观察不难发现，两个 www 位的二进制数相加，最多可能得到 w+1w+1w+1 位的结果，但数据类型只能存 www 位，需要丢掉最高位。也就是说，如果两个较大的数相加，我们可能会得到错误的结果，这就是溢出（Overflow）。可以用如下表达式表示加法：

s=Addw(u,v)=u+v mod 2ws=Add_w(u,v)=u+v\space mod\space 2^w s=Addw​(u,v)=u+v mod 2w

有符号整数和无符号整数的加法有着相同的位运算。有符号整数不仅会在正数上界有正溢出（Positive Overflow）的可能，变成一个负数；当两个负数相加时还有可能超过下界造成负溢出（Negative Overflow）变成一个很大的正数。

减法

减法可以看做一个数加上另一个数的相反数，与加法类似。

乘法

  • 两个 www 位的二进制数相乘，最多可能得到 2w2w2w 位，超出范围的位都要丢掉。
  • 无符号和有符号整数依旧有着相同的位运算。
  • 有符号整数乘法的Overflow同样可能会改变正负。

这里有一个很有用的小技巧：对一个数左移位相当于乘以2的次方
u << k 相当于 u∗2ku*2^ku∗2k (Signed,Unsigned都可用)

  • 位运算比乘法更快
  • 大多数编译器会自动检测，并帮你自动改写

浮点数（Floating Point）

浮点数在计算机中的存储方式与整数完全不同。浮点数要求能涵盖足够大的范围，同时又要保证小数的精度，以32位的float单精度型为例，最大等表示3.40282347E+38，最小的正数能表示1.17549435E-38，负数部分与正数部分对称。浮点数的范围比整数广得多，精度也比整数高。显然，要想达到这样的效果，绝对不可能像整数一样存储每一位数，必须另寻他法。

IEEE-754 浮点数

1985年IEEE-754发布，标准化了浮点数在计算机中的表示方式。计算机中所有的浮点数表示为以下数学公式：

(−1)S×M×2E(-1)^S\times M\times 2^E (−1)S×M×2E

  • S(Sign Bit 符号) 决定浮点数的正负
  • M(Significand 尾数) 表示一个介于1.0到2.0之间的数，对应下图的frac
  • E(Exponent 阶码) 决定小数点的位置，对应下图的exp

  • float 单精度浮点型exp长度为8-bits ，frac长度为23-bits
  • double双精度浮点型exp长度为11-bits，frac长度为52-bits

浮点数的阶码（Exp）

我们用阶码来控制小数点的左右移动，阶码可能是正数也可能是负数。以float为例，中间8位的exp并不完全对应公式中的E，真正的对应关系是 E=exp−biasE = exp - biasE=exp−bias ，其中 biasbiasbias 是一个偏移量，计算方式为 bias=2k−1−1bias = 2^{k-1}-1bias=2k−1−1 ，kkk 为exp的位数，8位的exp的偏移量为127。中间8位的exp等同于一个无符号整数，可表示1到254（exp为全1或全0为特殊情况，后面会提到），减去一个 biasbiasbias 刚好就是-126到127，完美表示正负。

同样是要表示正负，为什么exp不像有符号整数一样使用补码来表示呢？
可以观察到，当exp解析为无符号整数时，值越大，对应的浮点数一定越大，因此可以通过单独比较exp的位来快速比较两个浮点数的大小

浮点数的尾数（M）

我们的目标是将M表示为1到2之间的小数，即二进制数1.xxxx...x，既然小数点前都是1，我们可以直接丢掉，节省一位，只需表示小数点后的数。

  • 当frac全为0：M=1.0M = 1.0M=1.0
  • 当frac全为1：M=2.0−EM = 2.0 - \EpsilonM=2.0−E

exp的特殊值

浮点数的近似

浮点数无法表示所有现实中的有理数，因此在计算机存储浮点数或做浮点数计算前，会对浮点数做近似处理。计算机中浮点数的近似使用Nearest Even，即一般情况下使用四舍五入，如果处于两个整数之间，则近似到使得最低位为偶数**（对于二进制数来说就是0）**，比如将下列数近似到保留小数点后两位：

浮点数加法、减法和乘法

浮点数运算的一般步骤为：先计算实际值，如果有溢出则结果为正负无穷，然后取近似值。

需要特别注意的是，浮点数的加减法并不具有结合律，一个很大的数加上一个很小的数可能并不会改变这个数的值，因为越大的数之间的间隔也越大。在数学中成立的算式在计算机中就不一定成立了：

(3.14+1e10)−1e10=0(3.14 + 1e10) - 1e10 = 0 (3.14+1e10)−1e10=0

3.14+(1e10−1e10)=3.143.14 + (1e10 - 1e10) = 3.14 3.14+(1e10−1e10)=3.14

不同的计算顺序带来了完全不同的结果。如果其中一部分的计算结果有溢出，交换计算顺序之后同样也可能得到完全不同的结果。对于浮点数的乘法也是一样。

浮点数与整数之间的转换

在int float double之间转换，会改变数据的二进制表示bit level representation。

  • 浮点数->整数：朝原点近似（去尾法）
  • int->double：int有32位，double中的fracfracfrac部分有52位，double可以完全表示所有的int，不会丢失精度
  • int->float：近似处理，可能会丢失精度
